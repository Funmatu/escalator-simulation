# エスカレーター輸送効率シミュレーション (Escalator Efficiency Simulation)

エスカレーターにおいて「片側を空けて歩く」のと「両側で立ち止まる」のでは、どちらが輸送効率が良いのか？
物理単位（メートル、秒）に基づき、3D空間で検証できるシミュレーションです。

[👉 デモをブラウザで試す](https://funmatu.github.io/escalator-simulation/)

## 概要
このシミュレーターは、2本のエスカレーターを並走させ、以下のパターンを比較します。
* **左側（青）**: 両側立ち止まり (STANDING)
* **右側（赤）**: 片側歩行 (WALKING)

Three.jsを使用し、Webブラウザ上でリアルタイムに3D描画を行います。

## 機能
* **物理パラメータの調整**:
    * エスカレーターの長さ (m)
    * エスカレーターの速度 (m/s)
    * 歩行速度 (m/s)
    * 人が到着する間隔 (s)
* **時間操作**: 1倍速、10倍速、30倍速での高速シミュレーションが可能。
* **視点切り替え**: 全体俯瞰、または特定の人への一人称視点切り替え。
* **結果分析**: シミュレーション終了後、通過人数と効率比を算出。

## 操作方法
画面左上のコントロールパネルから操作します。

| ボタン/項目 | 説明 |
| :--- | :--- |
| **Speed (1x/10x/30x)** | シミュレーション速度を変更します |
| **Reset** | 設定を反映し、最初からやり直します |
| **一時停止** | シミュレーションを一時停止/再開します |
| **視点切替** | カメラモードを切り替えます |
| **各数値入力** | 長さや速度などのパラメータを変更します |
